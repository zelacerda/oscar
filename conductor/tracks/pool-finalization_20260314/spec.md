# Specification: Finalização de Bolões e Destaque do Vencedor

**Track ID:** pool-finalization_20260314
**Type:** Feature
**Created:** 2026-03-14
**Status:** Draft

## Summary

Permitir que o administrador finalize oficialmente os bolões quando todos os resultados estiverem registrados, e exibir o vencedor em destaque com parabéns na tela de ranking dos participantes.

## Context

O Oscar app já possui: registro de vencedores por categoria pelo admin, controle global de votações (toggle na home do admin), e tela de ranking com pontuação e posição. Falta o passo final — declarar oficialmente os vencedores dos bolões e celebrar o resultado.

## User Stories

1. Como administrador, quero finalizar os bolões quando todos os resultados estiverem registrados, para que os vencedores sejam oficialmente declarados.
2. Como participante, quero ver em destaque quem venceu o bolão com uma mensagem de parabéns, para que o resultado final fique claro e celebrado.

## Acceptance Criteria

- [ ] Quando todas as categorias tiverem vencedor registrado, aparece uma opção de "Finalizar Bolões" na tela do admin (estilo toggle similar ao Controle Global de Votações)
- [ ] O admin precisa confirmar a ação antes de finalizar (evitar clique acidental)
- [ ] Após finalização, os bolões ficam marcados como finalizados no banco
- [ ] Na tela de ranking do participante, quando o bolão está finalizado, o primeiro colocado aparece em destaque com mensagem de parabéns
- [ ] A opção de finalizar só aparece quando todas as categorias têm resultado

## Dependencies

- Código existente: tela de admin com registro de vencedores, tela de ranking dos participantes, sistema de pontuação

## Out of Scope

- Notificações push/email sobre a finalização
- Possibilidade de "desfinalizar" um bolão depois de confirmado
- Animações elaboradas ou confetti na tela de parabéns
- Tratamento especial de empate (se houver empate, todos os primeiros aparecem em destaque)

## Technical Notes

- Pool model precisa de um novo campo `isFinalized` (boolean)
- GlobalSettings pode ter um campo `allFinalized` para controle global
- Padrão visual do toggle similar ao Controle Global de Votações existente
- API precisa verificar se count(results) === count(categories) antes de permitir finalização
