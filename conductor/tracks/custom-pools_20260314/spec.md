# Specification: Bolões Customizáveis e Controle de Votação

**Track ID:** custom-pools_20260314
**Type:** Feature
**Created:** 2026-03-14
**Status:** Draft

## Summary

Permitir que qualquer usuário crie bolões com categorias e pesos customizáveis, e substituir o encerramento por horário por controle manual (dono do bolão encerra seu bolão, admin global encerra todos com override irrevogável).

## Context

Atualmente, apenas admins globais criam bolões via painel admin, com categorias fixas (tiers GOLD/SILVER/BRONZE/BASE) e encerramento por `lockDate` (data/hora). O sistema usa Prisma com PostgreSQL, NextAuth para auth, e a pontuação é calculada via `TIER_POINTS` fixo no ranking route.

## User Stories

- Como **participante**, quero criar meu próprio bolão com categorias e pesos personalizados para organizar apostas do jeito que eu quiser.
- Como **criador de um bolão**, quero encerrar as votações do meu bolão manualmente para ter controle sobre quando as apostas fecham.
- Como **admin global**, quero encerrar as votações de todos os bolões de uma vez para evitar fraudes durante a cerimônia.

## Acceptance Criteria

- [ ] Qualquer usuário logado pode criar um bolão (botão ativo para todos)
- [ ] Na criação do bolão, o usuário pode selecionar quais categorias incluir e definir pesos customizados (com os pesos atuais como default)
- [ ] O criador do bolão se torna administrador daquele bolão
- [ ] O dono do bolão pode encerrar as votações do seu bolão manualmente
- [ ] O admin global pode encerrar as votações de todos os bolões de uma vez (override irrevogável que nem donos de bolão podem reverter)
- [ ] O campo/lógica de encerramento por horário (`lockDate`) é removido

## Dependencies

Sistema existente de bolões, categorias, votação e permissões (Prisma schema, API routes, UI de pools).

## Out of Scope

- Sistema de notificações (avisar participantes que votação encerrou)
- Criação de novas categorias do Oscar (apenas seleção das existentes)
- Alteração do sistema de pontuação após o bolão ser criado
- Migração de bolões existentes para o novo formato (serão apagados)

## Technical Notes

- Schema atual: Pool tem `lockDate` (DateTime) e `adminId` — precisará de novos campos para lock manual e override global
- Scoring fixo em `TIER_POINTS` no ranking route — precisará ler pesos customizados do banco
- Bolões existentes serão apagados (sem necessidade de migração de dados)
